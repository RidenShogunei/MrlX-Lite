## 二、实验 A: 平滑奖励 GRPO

### 2.1 训练曲线 (来自 `grpo_v4` 日志)

```
Iter  1: Main R=0.537   Sub R=0.538
Iter  2: Main R=0.576   Sub R=0.551
Iter  3: Main R=0.593   Sub R=0.551
Iter  4: Main R=0.645   Sub R=0.553
Iter  7: Main R=0.706   Sub R=0.530   ★ Main 最佳
Iter  9: Main R=0.682   Sub R=0.560
Iter 15: Main R=0.698   Sub R=0.768   ★ Sub 最佳
Iter 19: Main R=0.553   Sub R=0.646   ⚠️ 开始退化
Iter 20: Main R=0.443   Sub R=0.624   ⬇
```

### 2.2 测试集评估 (来自 `analyze_results.py`)

| 阶段 | `<thinking>` | `[tool_call]` | `<result>` | 答案正确率 | Best-of-8 | 平均奖励 |
|------|:---:|:---:|:---:|:---:|:---:|:---:|
| 基础模型 | 100% | 98.8% | 100% | 65.0% | 100% | 0.516 |
| SFT后 | 100% | 97.5% | 100% | 60.0% | 100% | 0.460 |
| GRPO step 5 | 5.0% | 87.5% | 1.2% | 65.0% | 100% | 0.519 |
| GRPO step 10 | 0.0% | 5.0% | 0.0% | 76.2% | 90.0% | 0.545 |
| **GRPO step 15** | 0.0% | 81.2% | 0.0% | **80.0%** | **100%** | **0.617** |
| GRPO step 20 | 0.0% | 41.2% | 0.0% | **82.5%** | **100%** | 0.329 |

### 2.3 奖励分布变化

| 阶段 | 低奖励(0-0.3) | 中奖励(0.3-0.6) | 高奖励(0.6-1.0) |
|------|:---:|:---:|:---:|
| SFT后 | 51.2% | 20.0% | 28.7% |
| GRPO step 10 | 25.0% | 3.7% | **71.2%** |
| GRPO step 15 | 20.0% | 10.0% | **70.0%** |

### 2.4 关键发现

- ✅ 答案正确率: 60% → 82.5% (+22.5pp)
- ✅ 平均奖励: 0.460 → 0.617 (+34%)
- ❌ **格式完全崩溃**: `<thinking>` 100% → 0%, `<result>` 100% → 0%
- ❌ 模型学会"作弊": 丢掉标签、只输出数字拿高奖

**根因**: 平滑奖励中，答对数字奖励 0.60 太重，格式标签各 0.10 太轻。当GRPO采样到"无标签但数字对"的回复时，它获得 0.7 分，击败了"有标签但数字错"的回复(0.4分)。

## 三、实验 B: MrlX 二进制奖励 + SFT 初始化

### 3.1 奖励验证 (手动测试 10 题)

| 指标 | 结果 |
|------|------|
| 状态机格式通过率 | **0%** (0/10) |
| 答案正确率 | 100% (所有题目答案都对) |
| MrlX 奖励 | **全部 0.0** |

### 3.2 失败原因

SFT 训练使用的是 **中文 prompt**，SFT 模型输出的是中文散文风格：

```
模型实际输出: "分析：这是一个加法运算。计算表达式：82 + 15 = 97 结果：97"
MrlX状态机期望: <thinking>分析</thinking>[tool_call]82+15[/tool_call]<result>97</result>
```

→ **状态机一律判格式错误 → 奖励全 0 → 无训练信号 → 训练失败**

SFT 数据生成时只训练了"有 `<thinking>` `<result>` 标签"，并未训练严格的 XML 标签配对和状态机顺序。中文 prompt 下模型倾向于输出自然语言而非 XML。

## 四、实验 C: MrlX 二进制奖励 + 纯基础模型

**核心创新**: 
1. 纯基础模型(不加载SFT权重，避免中文散文习惯)
2. 英文 prompt + 强制 `<thinking>` 后缀（利用 Qwen 英文能力强）
3. 更严格的格式要求 + 零容忍二进制奖励

### 4.1 训练曲线

```
Iter  1:  ▏ R=0.128 满分= 6/50 (12%)    Sub R=0.320
Iter  2:  ▏ R=0.230 满分=11/50 (22%)    Sub R=0.543
Iter  3:  █ R=0.506 满分=24/50 (48%)    Sub R=0.560  ← 爆炸增长
Iter  4:  █ R=0.510 满分=24/50 (48%)    Sub R=0.526
Iter  5:  █ R=0.556 满分=27/50 (54%)    Sub R=0.641  ★ 保存
Iter  6:  █ R=0.594 满分=29/50 (58%)    Sub R=0.637  ★★ 最佳
Iter  7:  ▃ R=0.406 满分=20/50 (40%)    Sub R=0.626  ← 开始下降
Iter  8:  ▂ R=0.262 满分=13/50 (26%)    Sub R=0.373
Iter  9:  ▁ R=0.106 满分= 5/50 (10%)    Sub R=0.345
Iter 10:  ▁ R=0.120 满分= 6/50 (12%)    Sub R=0.331
Iter 11:  ▁ R=0.140 满分= 7/50 (14%)    Sub R=0.262  ← 不可逆崩溃
Iter 14:  ▁ R=0.040 满分= 2/50 ( 4%)    Sub R=...
```

### 4.2 阶段性分析

| 阶段 | 满分率 | 特点 |
|------|:---:|------|
| Iter 1-3 | 6 → 24 | **快速增长期**: 模型快速学会英文XML格式 |
| Iter 4-6 | 24 → 29 | **平稳优化期**: 在保持格式的同时优化答案 |
| Iter 7-8 | 29 → 13 | **拐点**: 某一轮采样到过多格式错误的回复 |
| Iter 9-14 | 13 → 2 | **不可逆崩溃**: 格式信号消失,模型彻底忘记 |

### 4.3 崩溃机制分析

```
正常训练:  Sample → 格式对(0.1/1.0) → SFT训练 → 格式越来越好
                             ↘ 格式错(0.0) → 跳过
                            
崩溃过程:  Sample → 格式错(0.0) → 跳过 → 无训练数据 
         → 下一轮格式更差 → 更多0.0 → 更少的训练数据
         → 恶性循环 → 模型彻底忘记格式
```

**核心问题**: 稀疏二元奖励在 **无 Trust Region (KL约束)** 的情况下, 无法防止模型"遗忘"。原始 MrlX 使用了 GRPO + KL 正则化保护格式, 我们的简化版缺少这个保护。

## 五、三组实验综合对比

### 5.1 核心指标

| 指标 | A-平滑奖励 | B-MrlX二元+SFT | C-MrlX二元+零初始 |
|------|:---:|:---:|:---:|
| 是否有训练信号 | ✅ | ❌ (全0) | ✅ (前6轮) |
| 答案正确率峰值 | **82.5%** | N/A | ~60% |
| 格式通过率峰值 | 100% | 0% | ~58% 
| 最佳奖励 | **0.617** | 0.0 | **0.594** |
| 格式保持 | ❌ (崩塌) | ❌ | ❌ (崩塌) |
| 训练稳定性 | ⚠️ (20轮后退化) | N/A | ❌ (7轮后崩溃) |
| SFT继承效果 | 有效(格式起点好) | 无效(中文不匹配) | N/A(没用SFT) |

### 5.2 奖励函数对比

| 特性 | 平滑奖励 | MrlX 二元奖励 |
|------|----------|---------------|
| 格式评分 | 部分奖励(0.1/tag) | **零容忍** (0.0) |
| 答案评分 | 部分奖励(0.6) | **全或无** (1.0/0.1) |
| 格式保护 | ❌ 太弱(格式=0.3 vs 答案=0.6) | ✅ 足够强(格式是必要条件) |
| 探索空间 | 大(部分奖励允许坏格式) | 小(坏格式直接0) |
| 适用场景 | 格式已固化的模型 | **格式需要强约束的模型** |

### 5.3 为什么 MrlX 二元奖励更好但在我们这里不稳定

**原始 MrlX 为什么稳定**:
1. **GRPO critic**：价值网络提供 baseline，减少方差
2. **KL 正则化**：约束策略更新幅度，防止格式遗忘
3. **批量训练**：每轮处理大量样本（几百条），信号密度大
4. **7B+ 模型**：更大的模型格式稳定性更好

**我们的简化版缺少**:
1. 无 Critic（使用 Best-of-N 替代）
2. 无 KL 正则化（只有 learning rate 限制）
3. 批量小（50题 × 4采样 = 200样本/轮）
4. 模型小（1.5B → 格式更容易漂移）

## 六、关键洞察

### 6.1 格式遗忘是GRPO最大敌人

```
实验 A: 格式 100% → 0%  (20轮)
实验 C: 满分 6 → 29 → 2  (6轮内崩)
```

每次都崩在同一个地方：**奖励信号变得稀疏时，模型迅速忘记格式**。

### 6.2 中文 SFT ≠ 中文格式

SFT 训练了"标签存在"但没有训练"严格XML顺序"。

```
SFT学会的:  <thinking>分析</thinking> 计算: 82+15=97 <result>97</result>  ← 标签在,但顺序乱
MrlX需要的:  <thinking>分析</thinking>[tool_call]82+15[/tool_call]<result>97</result>  ← 严格顺序
```

### 6.3 纯基础模型可能优于SFT初始化

当 prompt 与 SFT 分布不匹配时, SFT 起反作用:

```
基础模型 + 英文prompt → 6题满分 (12%) + 上升趋势
SFT模型 + 中文prompt   → 0题满分 (0%)  + 无训练信号
```

### 6.4 二元奖励的"悬崖效应"

| 奖励设计 | 格式错误 | 格式对+答案错 | 格式对+答案对 |
|----------|:---:|:---:|:---:|
| 平滑奖励 | 0.0~0.3 | 0.3~0.4 | 0.7~1.0 |
| MrlX二元 | **0.0** | **0.1** | **1.0** |

二元奖励下,格式错误的一步掉到谷底(=0.0), 无任何中间信号。这在有小批量、无KL保护的场景下极不稳定。

## 八、改进路线图

### 短期 (立即可行)

1. **混合奖励**: 平滑格式 + 二元答案
   ```python
   格式得分 = 0.3 × (has_thinking + has_tool_call + has_result)  # 保持平滑
   答案得分 = 0.7 × (1.0 if correct else 0.0)                    # 二元判断
   ```

2. **格式守卫**: 格式分 < 0.3 → reward = 0 (不管答案)
   ```python
   if format_score < 0.3:
       return 0.0
   ```

3. **提前停止**: 在训练信号密集时停止(实验C的iter 4-6)

### 中期 (需要代码改动)

4. **混合 Prompt**: 先用英文 prompt 训练格式 → 再切回中文微调
5. **增量数据集**: 分批加入新题，每批10题，避免剧烈波动
6. **Moving Average 奖励**: EMA(r) 替代即时奖励，平滑信号

### 长期 (需要更多基础设施)

7. **GRPO + KL 正则化**: 完整复制原始 MrlX 训练框架
8. **温度退火**: 训练早期高温度探索 → 晚期低温收敛
9. **Curriculum Learning**: 先训练简单加减法 → 再逐步加复杂类型
